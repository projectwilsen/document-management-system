## Deviations from Spec

1. **`backend/__init__.py` added (not in spec):** The brief specifies `from backend.config import settings` style imports throughout. For `pip install -e .` from inside `backend/` to expose `backend` as an importable top-level package, `backend/__init__.py` is required, and `pyproject.toml` needed `[tool.setuptools.packages.find]` with `where = [".."]` so setuptools discovers `backend` relative to the repo root.

2. **`pyproject.toml` build backend changed:** The brief specifies `setuptools.backends.legacy:build` which does not exist in setuptools 82.0.1 (Python 3.14.2 environment). Changed to the standard `setuptools.build_meta`.

3. **`backend/.env` not committed:** The brief instructs creating it for local testing but it contains credentials (even dummy ones). It is excluded from the commit. `.env.example` is committed instead.

4. **`backend/test.db` not committed:** Created by `alembic revision --autogenerate` using the SQLite URL. Not meaningful to commit.
